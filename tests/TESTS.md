# Тест-кейсы для csv_median_calculator

Каждый тест содержит папку `input/` с CSV-файлами и `expected/median_result.csv`
с ожидаемым выводом. Для проверки достаточно сравнить фактический вывод с ожидаемым:

```bash
diff expected/median_result.csv actual_output/median_result.csv
```

---

## TEST 1 — Базовый (пример из ТЗ)

**Что проверяет:** точность расчёта для нечётного и чётного числа элементов; одинаковый `receive_ts` в выходе — нормально.

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
1000;900;100.00000000;1.0;bid
2000;1900;101.00000000;1.0;ask
2000;1900;102.00000000;1.0;bid    ← тот же ts
3000;2900;103.00000000;1.0;ask
```



**Ожидаемый вывод:**
```
receive_ts;price_median
1000;100.00000000
2000;100.50000000
2000;101.00000000
3000;101.50000000
```

---

## TEST 2 — Все цены одинаковые

**Что проверяет:** медиана не меняется → в файл пишется только одна строка (самая первая).

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
1000;900;50.00000000;1.0;bid
2000;1900;50.00000000;1.0;ask
3000;2900;50.00000000;1.0;bid
4000;3900;50.00000000;1.0;ask
```

**Ожидаемый вывод:**
```
receive_ts;price_median
1000;50.00000000
```


---

## TEST 3 — Единственная запись

**Что проверяет:** корректная работа при минимальных данных (1 элемент).

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
9999;9000;123.45000000;1.0;bid
```

**Ожидаемый вывод:**
```
receive_ts;price_median
9999;123.45000000
```

## TEST 4 — Переключение формулы нечётное/чётное

**Что проверяет:** алгоритм правильно чередует формулы: один центральный элемент (нечётное) и среднее двух (чётное).

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
100;90;10.00000000;1.0;bid
200;190;20.00000000;1.0;ask
300;290;30.00000000;1.0;bid
400;390;40.00000000;1.0;ask
500;490;50.00000000;1.0;bid
```

**Ожидаемый вывод:**
```
receive_ts;price_median
100;10.00000000
200;15.00000000
300;20.00000000
400;25.00000000
500;30.00000000
```

## TEST 5 — Убывающая последовательность

**Что проверяет:** кучи корректно работают при добавлении элементов меньше ранее добавленных.

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
1;0;100.00000000;1.0;bid
2;1;90.00000000;1.0;ask
3;2;80.00000000;1.0;bid
4;3;70.00000000;1.0;ask
```


**Ожидаемый вывод:**
```
receive_ts;price_median
1;100.00000000
2;95.00000000
3;90.00000000
4;85.00000000
```

## TEST 6 — Одинаковый `receive_ts`, медиана меняется

**Что проверяет:** при одном и том же `ts` строки всё равно пишутся, если медиана изменилась (поведение из примера 4 ТЗ).

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
5000;4990;100.00000000;1.0;bid
5000;4990;200.00000000;1.0;ask   ← тот же ts
5000;4990;100.00000000;1.0;bid   ← тот же ts
```

**Ожидаемый вывод:**
```
receive_ts;price_median
5000;100.00000000
5000;150.00000000
5000;100.00000000
```

---

## TEST 7 — Одинаковый `receive_ts`, медиана НЕ меняется

**Что проверяет:** критерий записи — строго изменение медианы, а не `ts`. Запись: только одна строка, несмотря на 4 входных записи.

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
7000;6990;100.00000000;1.0;bid
7000;6990;100.00000000;1.0;ask   ← ts тот же, медиана та же
7000;6990;100.00000000;1.0;bid   ← ts тот же, медиана та же
8000;7990;200.00000000;1.0;ask
```


**Ожидаемый вывод:**
```
receive_ts;price_median
7000;100.00000000
```

---

## TEST 8 — Слияние двух файлов (level + trade)

**Что проверяет:** записи из двух файлов объединяются и сортируются по `receive_ts` перед обработкой. Файл `level.csv` содержит на одну колонку (`rebuild`) больше — это не должно мешать.

**Вход** (`input/level.csv`):
```
receive_ts;exchange_ts;price;quantity;side;rebuild
100;90;10.00000000;1.0;bid;0       ← ts=100
300;290;20.00000000;1.0;ask;0      ← ts=300
500;490;30.00000000;1.0;bid;1      ← ts=500
```

**Вход** (`input/trade.csv`):
```
receive_ts;exchange_ts;price;quantity;side
200;190;50.00000000;1.0;bid        ← ts=200
400;390;40.00000000;1.0;ask        ← ts=400
```


**Ожидаемый вывод:**
```
receive_ts;price_median
100;10.00000000
200;30.00000000
300;20.00000000
400;30.00000000
```

## Как запускать тесты

```bash
# Для каждого теста создать конфиг:
cat > /tmp/test_config.toml << EOF
[main]
input  = './tests/test1/input'
output = '/tmp/test1_output'
filename_mask = ['trade']
EOF

./build/csv_median_calculator --config /tmp/test_config.toml

# Сравнить с ожидаемым:
diff tests/test1/expected/median_result.csv /tmp/test1_output/median_result.csv
# Пустой вывод = тест прошёл
```

Для теста 8 указать `filename_mask = ['level', 'trade']`.
